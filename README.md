# Git Researcher

An AI-driven autonomous research system powered by Claude. It takes a research problem, breaks it down into sub-topics, and dispatches parallel research agents to investigate each angle — all orchestrated automatically.

## Architecture

```
┌─────────────────────────────────┐
│   research/problem.md           │  ← You define the research problem
│   research/supporting/          │  ← Add any supporting files
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│   Phase 1: Researcher Agent     │  Reads problem.md, asks clarifying
│   (Claude YOLO mode)            │  questions, produces:
│                                 │  → output/confirmed_statement.md
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│   Phase 2: Orchestrator Agent   │  Reads confirmed statement, generates
│   (Bun + Claude subprocesses)   │  research ideas:
│                                 │  → output/ideas/idea_001.md ...
│                                 │  → output/orchestrator_status.md
└──────────────┬──────────────────┘
               │  spins up N parallel
               ▼
┌─────────────────────────────────┐
│   Phase 3: Sub-Agents           │  Each gets one idea + the research
│   (Claude YOLO mode, parallel)  │  problem. Investigates, calculates
│                                 │  matching metrics, returns results:
│                                 │  → output/results/result_001.md ...
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│   Orchestrator updates status   │  Aggregates results, updates
│                                 │  → output/orchestrator_status.md
│                                 │  → output/final_report.md
└─────────────────────────────────┘
```

## Prerequisites

- [Bun](https://bun.sh/) runtime
- [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) installed and authenticated

## Quick Start

1. **Define your research problem:**
   ```bash
   cp templates/problem_template.md research/problem.md
   # Edit research/problem.md with your research question
   ```

2. **Add supporting files** (optional):
   ```bash
   # Place any relevant documents in research/supporting/
   ```

3. **Run the research pipeline:**
   ```bash
   bun run start
   ```

   This will execute all three phases automatically.

   Or run phases individually:
   ```bash
   bun run phase1    # Researcher agent only
   bun run phase2    # Orchestrator + sub-agents
   ```

## How It Works

1. **Phase 1 — Researcher Agent**: Claude reads your `research/problem.md`, verifies it understands the problem, identifies what is already known, and produces a `confirmed_statement.md` with key metrics and knowledge boundaries.

2. **Phase 2 — Orchestrator**: A Bun process reads the confirmed statement and spawns Claude to generate research ideas. Each idea gets its own markdown file. Then the orchestrator spawns parallel sub-agents.

3. **Phase 3 — Sub-Agents**: Each sub-agent receives its assigned research idea plus the original problem context. It investigates using web search, calculates relevance/matching metrics, and writes results back. The orchestrator monitors progress and updates `orchestrator_status.md`.

All Claude invocations run in **YOLO mode** (`--dangerously-skip-permissions`) for fully autonomous operation.
